Pendahuluan

Konteks Proyek

Anda adalah satu dari 16 developer full-stack yang ditugaskan untuk membuat aplikasi "Katalog Interaktif" mini secara individual. Setiap developer akan mendapatkan satu tema unik dari daftar yang disediakan.

Tujuan Proyek

Membangun aplikasi web full-stack yang berfungsi penuh, menampilkan katalog produk sesuai dengan tema yang ditugaskan. Aplikasi harus memiliki fitur frontend (React) dan backend (Node.js) untuk mengelola data, termasuk fungsi CRUD.

Teknologi Inti (Wajib)

Node.js Express.js Prisma React React Context API Axios

Struktur Pengerjaan

Setiap developer harus mengimplementasikan struktur berikut untuk aplikasi mereka. Gunakan tab di bawah untuk beralih antara spesifikasi Backend dan Frontend.

1. Inisiasi Proyek & Server

  • Siapkan server Express dasar.
  • Aktifkan CORS dan `express.json()`.
  • Siapkan file `.env` untuk variabel lingkungan (misal: `DATABASE_URL`, `PORT`).

2. Skema Database (Prisma)

Desain `schema.prisma` dengan model `Product` dan `Category`. Jalankan migrasi.

Contoh Skema Minimal:

// schema.prisma
model Category {
  id        Int       @id @default(autoincrement())
  name      String    @unique
  products  Product[]
}

model Product {
  id          Int       @id @default(autoincrement())
  name        String
  price       Int
  description String?
  imageUrl    String?
  category    Category  @relation(fields: [categoryId], references: [id])
  categoryId  Int
}

// Opsional untuk fitur Cart/Order
model Order {
  id           Int       @id @default(autoincrement())
  customerName String
  totalPrice   Int
  items        OrderItem[]
}

model OrderItem {
  id        Int     @id @default(autoincrement())
  order     Order   @relation(fields: [orderId], references: [id])
  orderId   Int
  productId Int     
  quantity  Int
  price     Int     
}

3. Implementasi API (CRUD)

Buat RESTful API untuk mengelola produk (dan opsional Kategori).

POST /api/products GET /api/products GET /api/products/:id PUT /api/products/:id DELETE /api/products/:id

4. Implementasi API (Cart/Order)

Menerima array item dari cart dan menyimpannya sebagai `Order` baru.

POST /api/checkout

1. Struktur Halaman (Routing)

Gunakan `react-router-dom` untuk navigasi halaman.

/ /cart /admin

2. Katalog Interaktif (Homepage)

  • *Fetch* data dari `GET /api/products`.
  • Tampilkan semua produk dalam bentuk *card*.
  • Sediakan tombol "Tambah ke Keranjang" di setiap *card*.

3. Fungsi Keranjang (Cart)

  • Gunakan **Context API** untuk mengelola *state* keranjang (`cartItems`).
  • Halaman `/cart` menampilkan semua item, jumlah, dan total harga.
  • Implementasikan fungsi tambah/kurang/hapus item.
  • Buat tombol "Checkout" yang mengirim data ke `POST /api/checkout`.

4. Fungsi CRUD (Halaman Admin)

  • Buat halaman `/admin` (tidak perlu *login* untuk studi kasus ini).
  • Sediakan **Formulir** untuk **Create** dan **Update** produk.
  • Tampilkan **Tabel** semua produk dengan tombol **Edit** dan **Delete**.

Penugasan Tema (16 Developer)

Setiap developer harus mengadaptasi soal di atas agar sesuai dengan tema (domain bisnis) yang ditugaskan. Gunakan filter atau pencarian di bawah untuk menjelajahi 16 tugas.

Kriteria Penilaian

  • Kelengkapan Fungsionalitas Backend: Server, Skema, API CRUD, dan API Checkout berfungsi sesuai spesifikasi.
  • Kelengkapan Fungsionalitas Frontend: Routing, Tampilan Katalog, *State Cart*, dan Halaman Admin CRUD berfungsi sesuai spesifikasi.
  • Kesesuaian Tema: Implementasi dan data *dummy* relevan dengan tema yang ditugaskan.
  • Kualitas Kode: Kode bersih, terstruktur, dan mudah dibaca.
